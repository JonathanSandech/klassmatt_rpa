O que e um SIN?
SIN = Solicitacao de Inclusao de Novo item. Cada linha da planilha Excel e um SIN com um numero unico (ex: 474726). O bot abre cada SIN no Klassmatt e preenche os dados.

Fluxo de Processamento por Item

Para cada SIN da planilha Excel, o bot executa estes passos em sequencia:

Buscar SIN na Worklist
pages/worklist.py + pages/item.py
Navega ate a Worklist, aplica filtro "Todas as Solicitacoes", busca o numero do SIN no grid de resultados (#DIVResultado).
Atuar no Item / Criar Item
pages/item.py
Clica "Atuar no Item" para abrir o formulario. Se o item ainda nao existe, clica "Criar Item" → "Finalizar" → "Salvar" → "Sim". Verifica se o status e FINALIZACAO — se nao for, pula o item.
UNSPSC (Classificacao)
pages/classifications.py
Abre a aba Classificacoes, clica no botao UNSPSC (#ibutUNSPSC), preenche o codigo no popup, pesquisa, seleciona o resultado e salva. Se o codigo nao existir na base, loga warning.
NCM/TIPI (Fiscal)
pages/fiscal.py
Abre a aba Fiscal, formata o NCM de 73181500 para 7318.15.00, preenche o campo #txtNCMTIPI, pressiona Tab para validar. Se o Klassmatt rejeitar, limpa o campo e loga warning.
Referencia (Empresa + Part Number)
pages/references.py
Abre a aba Referencias. Verifica se ja existe referencia. Se nao, clica #iButAddRef para adicionar, preenche empresa (com autocomplete) e part number, salva. Trata warning de "Referencia igual".
Relacionamento (Codigo Antigo)
pages/relationships.py
Abre a aba Relacionamentos. Adiciona relacao tipo "CODIGO ANTIGO" com status "ATIVO ERP" e comentario "ZBRA". Verifica duplicatas antes de inserir.
Upload de Midias (Documentos PDF)
pages/media.py
Abre a aba Midias (abre em nova aba do browser). Para cada documento: clica "Adicionar Midia", faz upload do arquivo, preenche titulo e salva. Problema conhecido: link "Adicionar Midia" desaparece apos 5+ uploads consecutivos.
Salvar Estado
pages/references.py (butSalvar)
Persiste as alteracoes de referencias/relacionamentos clicando butSalvar. Testa navegacao para confirmar que o dirty state foi limpo.
Validacao SAP (Descricao)
pages/descriptions.py
Verifica se a descricao SAP tem mais de 40 caracteres. Se sim, desmarca "Exibe D2" na referencia para nao mostrar descricao longa.
Alterar PDM (Padrao de Material)
pages/descriptions.py
Navega para Descricoes → Editar Descricao (DescricaoV3.aspx). Pesquisa o codigo PDM, seleciona a categoria e clica "Definir Padrao". Isso habilita a tabela de atributos tecnicos.
Preencher Atributos Tecnicos
pages/attributes.py
Na pagina DescricaoV3, percorre a tabela dgDadosTecnicos. Para cada atributo: se "N/A" marca o checkbox, senao abre o popup da arvore taxonomica (Dt_EditaArvore.aspx), navega pela letra do alfabeto e seleciona o valor. Clica "Finalizar" para salvar tudo.
Voltar para Worklist
browser.py + pages/worklist.py
Navega de volta: Principal → Worklist → Re-aplica filtro. Espera pesquisar() estar disponivel antes de processar o proximo SIN. Delay de 5s entre itens.
Retry Logic
Se algum passo falha, o bot tenta ate 3 vezes com backoff. Se a sessao expirar (KlassmattSessionError), reinicia o browser inteiro. O progresso e salvo em progress.json a cada item.

Verify & Fix

O verify_and_fix.py e o script de pos-processamento. Ele abre cada SIN no Klassmatt, compara os dados preenchidos com o que esta no Excel, e tenta corrigir divergencias automaticamente.

O que ele verifica

  • UNSPSC — codigo de classificacao
  • NCM — codigo fiscal (formato XXXX.XX.XX)
  • Referencia — empresa + part number
  • Relacionamento — codigo antigo
  • PDM — padrao de material
  • Atributos — ate 30 atributos tecnicos
  • Midias — quantidade de documentos uploaded

Como usar

python verify_and_fix.py                         # todos os SINs da planilha
python verify_and_fix.py 474284 474291           # SINs especificos
python verify_and_fix.py --file=lista.txt        # SINs de um arquivo (um por linha)
python verify_and_fix.py --only-divergent        # re-processa somente os que falharam no report
python verify_and_fix.py --verify-only           # so verifica, nao corrige nada

run_verify.bat                                   # com auto-restart em caso de crash
run_verify.bat --verify-only                     # verify only com auto-restart

Armadilhas e Padroes Importantes

Overlay div1 — O problema #1
O Klassmatt tem um <div id="div1"> (painel de descricao) que fica invisivel mas por cima de tabs e botoes. Cliques do Playwright sao interceptados silenciosamente — o click parece funcionar, mas nada acontece.

Solucao: Sempre chamar hide_overlays(page) ANTES de clicar em tabs. Se o click falhar, usar page.evaluate("el => el.click()") como fallback JS. A funcao safe_click() ja faz isso automaticamente.
IDs ASP.NET sao diferentes do name
O atributo name usa hierarquia completa (ctl00$Body$tabFiscal$txtNCMTIPI), mas o id renderizado e curto (txtNCMTIPI). Sempre usar o id para seletores CSS. IDs sao case-sensitive (Imagebutton7 != imagebutton7).
Rate Limiting do Klassmatt
Navegar rapido demais causa "Ocorreu uma excecao". O bot usa delay de 5s entre itens e espera pesquisar() estar definida antes de buscar o proximo SIN.
Idempotencia obrigatoria
Cada passo precisa verificar se ja foi feito antes de tentar fazer. Itens podem ser reprocessados apos erros. Exemplos: verificar contagem de referencias antes de adicionar, verificar contagem de midias na aba antes de upload, verificar se NCM ja esta preenchido.
Sessao expira
O Klassmatt desconecta apos inatividade. O bot detecta isso via verificar_sessao() e pausa para re-login manual. O playwright_profile/ persiste cookies, mas a sessao do servidor expira.
Tabs abrem em nova janela
A aba "Midias" abre em uma nova aba do browser (via OpenNewTab() JS). O bot detecta isso checando page.context.pages. O botao "Fechar" da aba Midias (cmdFechar) destrói a pagina imediatamente — precisa de try/except.
dgDadosTecnicos fica em DescricaoV3
A tabela de atributos tecnicos NAO fica na pagina principal do item (ITEM_Edita.aspx). Fica em ITEM_Edita_DescricaoV3.aspx, acessivel via Descricoes → Editar Descricao. So aparece apos definir um PDM.
